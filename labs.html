Lab Challenges

Hands-on cybersecurity challenges demonstrating practical skills in various security domains.

Web Application SQL Injection (Web Security)

Problem Statement: A vulnerable web application with a login form was susceptible to SQL injection. The task was to bypass authentication and retrieve sensitive data.

Approach: Tested manually with payloads, confirmed vulnerability, then automated extraction with SQLmap.

Tools Used: Burp Suite, SQLmap, Browser DevTools

Key Lessons:

  • Importance of parameterized queries in web applications
  • Manual and automated SQL injection testing
  • Secure coding practices to mitigate SQLi

Network Traffic Analysis (Network Security)

Problem Statement: Analyzed a suspicious PCAP file to detect malicious traffic and exfiltrated data.

Approach: Inspected packets in Wireshark, detected DNS tunneling, and extracted hidden payloads.

Tools Used: Wireshark, Tshark, Python scripts

Key Lessons:

  • Normal vs. abnormal network patterns
  • DNS-based data exfiltration techniques
  • Importance of continuous monitoring

Reverse Engineering CrackMe (Binary Exploitation)

Problem Statement: A binary executable required a secret password to unlock functionality.

Approach: Decompiled binary in Ghidra, traced validation logic, retrieved correct password.

Tools Used: Ghidra, GDB, Radare2

Key Lessons:

  • Fundamentals of reverse engineering
  • Debugging binaries for logic flow
  • Risks of hardcoding credentials

Privilege Escalation (Linux) (System Security)

Problem Statement: Gained low-privileged shell on a vulnerable machine but required root access.

Approach: Enumerated SUID binaries, leveraged misconfigured scripts to escalate privileges.

Tools Used: LinPEAS, GTFOBins, Linux commands

Key Lessons:

  • Privilege escalation vectors on Linux
  • Importance of patching misconfigurations
  • Realistic attacker post-exploitation tactics

Malware Analysis Lab (Malware & Threat Analysis)

Problem Statement: Analyzed a suspicious Windows executable suspected to be malware.

Approach: Performed static analysis with strings and PE tools, followed by sandbox execution to observe behavior.

Tools Used: Strings, PEStudio, Any.Run Sandbox

Key Lessons:

  • Extracting IOCs from malware samples
  • Difference between static and dynamic analysis
  • Importance of sandbox environments for safety

Brute Force & Password Cracking (Offensive Security)

Problem Statement: Faced a weakly protected login portal requiring password discovery.

Approach: Captured login requests, then used Hydra and JohnTheRipper to brute force weak credentials.

Tools Used: Hydra, JohnTheRipper, Burp Suite

Key Lessons:

  • Risks of weak password policies
  • Rate limiting and lockout mechanisms
  • Strong password management strategies

Incident Response Simulation (Blue Teaming)

Problem Statement: Investigated a simulated attack on a corporate network environment.

Approach: Reviewed Splunk SIEM alerts, traced malicious activity, contained affected endpoints.

Tools Used: Splunk, Sysinternals Suite, Event Logs

Key Lessons:

  • Incident detection and triage
  • Log correlation for threat hunting
  • Responding quickly to limit damage

Threat Intelligence & Hunting (Threat Intelligence)

Problem Statement: Tasked with investigating a targeted phishing campaign and mapping attacker TTPs.

Approach: Collected OSINT data, extracted IOCs from email headers, cross-referenced with VirusTotal and MITRE ATT&CK, and developed a threat profile.

Tools Used: OSINT Framework, VirusTotal, MISP, MITRE ATT&CK Navigator

Key Lessons:

  • How to extract actionable intelligence from raw indicators
  • Correlating attacker behaviors with MITRE ATT&CK techniques
  • Importance of proactive threat hunting to reduce dwell time
