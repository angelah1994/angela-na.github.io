Attack Analysis & Exploitation – H4cked (TryHackMe)

Overview: Analyzed captured network traffic to investigate a compromised Linux machine, then reproduced the attack chain to validate findings and successfully retrieve the system flag.

Phase 1 – Network Traffic Analysis (Wireshark):

Loaded and examined packet capture (PCAP) files in Wireshark to identify suspicious activity. Detected repeated FTP authentication attempts and observed credential brute-force behavior targeting a specific user account.

  • Filtered FTP traffic
  • Identified failed and successful login attempts
  • Determined attack vector from captured packets

Phase 2 – Reconnaissance:

Performed a full TCP port scan to identify open services and running versions.

Command: nmap -A -T4 -Pn -p- [target IP]

Phase 3 – Credential Brute Force:

Reproduced the attack using Hydra to brute-force FTP credentials with the rockyou wordlist.

Command: hydra -l jenny -P /usr/share/wordlists/rockyou.txt [target IP] ftp

Phase 4 – Initial Access & Payload Deployment:

Logged into FTP with recovered credentials, downloaded relevant files, and uploaded a reverse shell payload (shell.php).

  • Used get to retrieve files
  • Used put to upload backdoor

Phase 5 – Reverse Shell & Stabilization:

Started a Netcat listener before triggering the payload:

Command: nc -lvnp 4444

Upgraded to an interactive TTY shell:

Command: python3 -c 'import pty; pty.spawn("/bin/bash")'

Phase 6 – Privilege Enumeration:

Checked sudo permissions to determine available elevated access.

Command: sudo -l

Phase 7 – Flag Retrieval:

Navigated the file system and successfully located flag.txt, completing the full exploitation lifecycle.

Tools Used: Wireshark, Nmap, Hydra, FTP, Netcat, Linux CLI, Python (TTY upgrade)

  • Packet-level brute-force detection and analysis
  • Full attack chain reproduction
  • Reverse shell deployment and stabilization
  • Privilege enumeration and post-exploitation navigation
  • Understanding attacker methodology from investigation to exploitation
